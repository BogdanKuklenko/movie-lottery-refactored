🎯 STEP-BY-STEP GUIDE: Fix Render Start Command
================================================

YOUR MISSION: Tell Render to use gunicorn_config.py

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 1: Open Render Dashboard
------------------------------
Go to: https://dashboard.render.com


STEP 2: Find Your Service
--------------------------
Look for: "movie-lottery" (or whatever you named it)
Click on it


STEP 3: Click Settings
-----------------------
Top right corner, click: "Settings" button


STEP 4: Scroll to Start Command
--------------------------------
Look for section called: "Start Command"
Current value probably shows:
   gunicorn "movie_lottery:create_app()"

Change it to:
   gunicorn --config gunicorn_config.py "movie_lottery:create_app()"


STEP 5: Save Changes
---------------------
Click: "Save Changes" button at bottom
Render will automatically redeploy (takes 2-3 minutes)


STEP 6: Watch the Logs
-----------------------
Click: "Logs" tab
Watch for:
   ✅ [INFO] Starting gunicorn 23.0.0
   ✅ [INFO] Listening at: http://0.0.0.0:10000
   ✅ [INFO] Booting worker with pid: XXX
   ✅ ==> Your service is live 🎉

NO MORE TIMEOUT ERRORS! 🎉


STEP 7: Initialize Database (ONE TIME ONLY)
--------------------------------------------
After successful deployment, run:
   python init_db.py

Or visit this URL in your browser:
   https://your-app-name.onrender.com/init-db/super-secret-key-for-db-init-12345


STEP 8: Test Your App
----------------------
Visit your app URL:
   https://your-app-name.onrender.com

Try:
   - Home page loads ✅
   - Create a lottery ✅
   - View history ✅
   - View library ✅


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

TROUBLESHOOTING:

If still timing out after 5 minutes:
   → Check Environment Variables
   → Make sure DATABASE_URL is set
   → Consider upgrading to Starter plan ($7/month)

If you see "command not found":
   → Try: gunicorn -c gunicorn_config.py "movie_lottery:create_app()"
   → Or use inline: gunicorn --timeout 300 --workers 1 "movie_lottery:create_app()"

If database errors:
   → Run init_db.py to create tables
   → Check DATABASE_URL format: postgresql://...

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

BEFORE vs AFTER:
================

BEFORE (broken):
   Start Command: gunicorn "movie_lottery:create_app()"
   Timeout: 30 seconds → ❌ WORKER TIMEOUT
   
AFTER (fixed):
   Start Command: gunicorn --config gunicorn_config.py "movie_lottery:create_app()"
   Timeout: 300 seconds → ✅ SUCCESS
