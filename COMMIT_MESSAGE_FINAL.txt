fix: Усиление проверок qBittorrent credentials для предотвращения timeout

Добавлена более строгая проверка переменных окружения qBittorrent перед попытками подключения.

Изменения:
- Проверка что QBIT_* переменные не только существуют, но и НЕ пустые
- Проверка что строковые значения не содержат только пробелы
- Ранний выход БЕЗ попыток подключения если credentials некорректны

Проблема:
- Worker продолжал пытаться подключиться к qBittorrent даже с пустыми credentials
- Множественные timeout'ы (5+ попыток) при запуске worker'а
- Worker TIMEOUT через 30 секунд

Решение:
- Если QBIT_HOST/PORT/USERNAME/PASSWORD отсутствуют или пусты
  → Немедленный возврат с qbittorrent_available=false
  → БЕЗ попыток подключения (0 секунд)
  → Worker запускается мгновенно

Результат:
- Приложение запускается за < 1 секунды даже без qBittorrent
- Нет множественных timeout'ов при старте
- Graceful degradation - сайт работает без qBittorrent

Требуется также:
- Удалить QBIT_* переменные на Render (или установить пустыми)
- Обновить Start Command: gunicorn --timeout 300 ...
